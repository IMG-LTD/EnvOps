# EnvOps v0.0.2 Release Notes

## Highlights

- `frontend/` 与 `backend/` 已形成统一的 `v0.0.2` 主工作区基线。
- 动态路由登录流程已经对接真实后端接口。
- 资产、检测、应用、发布任务、任务中心已形成可联调主链路。
- `deploy/task` 页面已支持列表、详情、主机明细、执行日志，以及 `execute` / `retry` / `rollback` / `cancel`。
- 仓库根 `README.md`、项目说明、开发说明、用户手册已统一更新到 `v0.0.2` 口径。

## Included scope

当前版本纳入说明与交付范围的内容包括：

- `backend/`：Spring Boot 3.3 多模块后端，包含 `envops-boot` 启动入口、统一响应、安全配置与本地种子数据。
- `frontend/`：Vue 3 + Vite + TypeScript 前端控制台，包含动态路由登录、请求层、资产中心、检测中心、应用中心、安装发布与任务中心主链路。
- `docs/`：
  - `envops-v0.0.2-项目详细说明.md`
  - `envops-v0.0.2-开发技术说明.md`
  - `envops-v0.0.2-用户操作手册.md`
- 根 `README.md`：包含当前版本概览、Quick Start、文档入口、默认账号和限制说明。
- `release/`：当前版本发布说明与发布检查清单。

## Quick start

```bash
cd backend && \
ENVOPS_SECURITY_TOKEN_SECRET=change-this-token-secret-2026-1234 \
ENVOPS_CREDENTIAL_PROTECTION_SECRET=change-this-credential-secret-2026 \
mvn -pl envops-boot spring-boot:run
```

```bash
cd frontend && pnpm install && pnpm dev
```

默认体验账号：`envops-admin / EnvOps@123`

补充说明：

- 后端默认地址：`http://127.0.0.1:8080`
- 前端默认地址：`http://localhost:9527`
- 两个后端安全相关环境变量都要求至少 32 个字符
- 后端启动时自动加载 `schema.sql` 与 `data.sql`

## Verification summary

本次版本说明所对应的可验证项包括：

- `cd backend && mvn test`
- `cd frontend && pnpm test:unit`
- `cd frontend && pnpm typecheck`
- 使用默认账号登录并验证动态菜单加载
- 打开 `deploy/task` 页面查看示例任务详情、主机明细与执行日志

## Known limitations

- `home`、`system/user`、`traffic/controller` 当前仍以展示页为主。
- 前端当前没有“新建发布任务”“审批通过/拒绝”的页面入口。
- `envops-exec` 当前仍是模块骨架。
- `frontend/.env.prod` 的 `VITE_SERVICE_BASE_URL` 仍为空，生产部署前需明确 API 接入方式。
- 流量控制当前更偏目录展示与能力预留，不应视为完整生产化流量系统。
