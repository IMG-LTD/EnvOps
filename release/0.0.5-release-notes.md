# EnvOps v0.0.5 Release Notes

## Highlights

1. Traffic 页面与接口不再停留在 skeleton / not-ready 口径，已收敛为最小真实切流 MVP。
2. 当前 Traffic 只支持一个插件通路：`REST`。
3. 当前 Traffic 只支持一个策略：`weighted_routing`。
4. `preview`、`apply`、`rollback` 三个动作都会真实调用外部 REST 流量服务。
5. 只有外部调用成功，EnvOps 才更新策略状态与 `rollbackToken`；外部服务不可用时统一返回 `502 Bad Gateway`。
6. `apply` 成功路径强制要求外部服务返回 `rollbackToken`，避免把不可回滚状态写成成功。
7. 前端 Traffic 页面会明确提示 MVP 范围、区分支持与不支持的记录，并展示最近一次动作成功或失败反馈。
8. 数据库资源管理继续保持真实连通性检测能力，已支持 `mysql`、`postgresql`、`oracle`、`sqlserver`、`mongodb`、`redis` 六类主流数据库，覆盖单条、已选、当前页、全部筛选结果四种检测入口。
9. 数据库资源中的 `warning` 连通性状态仍为人工维护；编辑密码时留空表示沿用已保存密文，响应不回显密码。

## Scope Boundary

1. Traffic MVP 只覆盖 `REST` + `weighted_routing` + `preview/apply/rollback` 真闭环。
2. 当前不扩展多插件、多策略矩阵、审批体系重构、批量切流、高级编排与灰度报表。
3. `rollback` 只对已有 `rollbackToken` 的记录开放。
4. 页面上不满足支持条件的记录会保持禁用，并明确展示原因。
5. 数据库资源仍只支持基础直连，不支持 SSL、连接串、MongoDB 副本集、Redis Sentinel / Cluster 等高级连接能力。
