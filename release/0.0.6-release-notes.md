# EnvOps 0.0.6 Release Notes

## Summary

- Task Center 已从仅面向 Deploy 的视图收敛为最小真实统一任务中心。
- 当前统一纳入三类任务：Deploy、数据库连通性检测、Traffic 动作。
- 前端提供统一列表、最小筛选、轻量详情抽屉，以及“查看原始详情”深链返回原模块。
- Deploy 历史通过统一投影补录；数据库检测与 Traffic 动作从 0.0.6 起记录新增任务；数据库批量检测按一批一条记录。

## Scope

### Included in 0.0.6

- 三类统一任务类型：`deploy`、`database_connectivity`、`traffic_action`
- 四类统一状态口径：`pending`、`running`、`success`、`failed`
- Deploy 历史任务补录到统一任务投影
- 数据库单次检测与批量检测新增统一任务记录
- Traffic `preview` / `apply` / `rollback` 动作新增统一任务记录
- 任务列表支持关键词、任务类型、状态、开始时间范围筛选
- 详情交互保持轻量抽屉承载，并通过源路由深链回原模块

### Not included in 0.0.6

- 任务中心内直接重试、取消或多任务编排
- 数据库与 Traffic 的既有历史任务补录
- 新的执行器抽象层
- 超出当前源模块能力的详情整合

## User-facing behavior

1. 在 Task Center 中可统一查看 Deploy、数据库检测、Traffic 动作三类任务。
2. 列表可按关键词、任务类型、状态、开始时间范围筛选。
3. 点击任务先打开轻量详情抽屉，查看基础信息、摘要、失败原因或批量汇总信息。
4. 如需更深内容，可继续点击“查看原始详情”回到原模块页面。

## Technical notes

- Deploy 历史由统一投影在当前版本补录，保证旧 Deploy 任务可进入统一列表。
- 数据库检测与 Traffic 动作仅从当前版本开始记录新增统一任务，不回补既有历史。
- 数据库批量检测按一批一条记录，避免把同一批检测拆成多条子任务。
- 数据库与 Traffic 当前深链回页面级入口：`/asset/database`、`/traffic/controller`；Deploy 深链回 `/deploy/task?taskId=...`。
