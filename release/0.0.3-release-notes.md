# EnvOps v0.0.3 Release Notes

## Highlights

- `asset/host` 页面已从只读列表补齐为“主机纳管 + 列表刷新 + 指标详情跳转”。
- `monitor/detect-task` 页面已支持检测任务创建与手动执行。
- `deploy/task` 页面已支持任务创建、审批通过/驳回、详情、主机明细、执行日志以及任务动作处置。
- `system/user` 页面已从静态展示补齐为“真实列表 + 新建用户 + 编辑用户”。
- `traffic/controller` 页面已支持 `preview` / `apply` / `rollback`，并对接真实后端动作接口。
- README、项目说明、开发说明、用户手册已同步更新到 `v0.0.3` 最新口径。

## Included scope

当前版本纳入说明与交付范围的内容包括：

- `backend/`：Spring Boot 3.3 多模块后端，包含登录鉴权、资产、检测、应用、发布任务、任务中心、系统用户、流量策略等模块。
- `frontend/`：Vue 3 + Vite + TypeScript 前端控制台，包含动态路由登录、资产中心、检测中心、应用中心、安装发布、任务中心、系统管理与流量控制页面。
- `docs/`：
  - `envops-v0.0.3-项目详细说明.md`
  - `envops-v0.0.3-开发技术说明.md`
  - `envops-v0.0.3-用户操作手册.md`
- 根 `README.md`：包含当前版本概览、Quick Start、文档入口、默认账号和限制说明。
- `release/0.0.3-release-notes.md`：当前版本变更记录。

## Key changes by module

### 资产中心

- 前端新增 `CreateHostParams` 类型定义。
- 前端新增 `fetchCreateAssetHost` API 封装。
- `asset/host` 页面新增主机纳管表单、必填校验、成功提示和列表刷新。
- 已有监控快照的主机仍可从主机列表直接跳转指标详情。

### 检测中心

- 检测任务页面已支持任务创建。
- 检测任务页面已支持手动执行已有任务。
- 页面创建动作依赖资产主机列表作为目标主机选择来源。

### 安装发布

- `deploy/task` 页面新增创建任务抽屉。
- 页面已对接 `POST /api/deploy/tasks`。
- 页面已支持审批通过 / 驳回抽屉，并对接对应后端接口。
- 发布任务详情继续支持主机明细、执行日志、自动刷新与任务动作处置。

### 系统管理

- 后端新增 `GET /api/system/users`、`POST /api/system/users`、`PUT /api/system/users/{id}`。
- `UserApplicationService` 新增用户创建、更新、手机号校验、角色绑定、唯一性校验等逻辑。
- 前端新增 `fetchCreateSystemUser`、`fetchUpdateSystemUser`。
- `system/user` 页面新增抽屉式新建与编辑表单。
- 页面支持团队、登录方式、状态、角色维护。

### 流量控制

- 后端流量策略已落到 `traffic_policy` 持久化表。
- 前端新增 `preview` / `apply` / `rollback` 请求封装。
- 流量规则页已从目录展示升级为可操作页面。
- 回滚按钮可用性已与 plugin 能力和回滚令牌状态联动。
- `TrafficPlugin` 的能力位与 `pluginStatus` 已在文档中明确区分。

### Plugin / Executor 边界

- `TrafficPlugin` 负责流量动作，当前包含 `NGINX` / `REST` 两个 skeleton adapter。
- `RemoteExecutor` 负责发布任务远程执行，当前 `SshRemoteExecutor` 状态为 `READY`。
- 文档已明确 `supportsPreview` / `supportsApply` / `supportsRollback` 是能力位，不等于真实系统已接通。

### 文档

- README 已切换到 `v0.0.3` 最新能力描述。
- `v0.0.3` 项目详细说明、开发技术说明、用户操作手册已全部同步到最新代码口径。
- 历史 `v0.0.2` 文档继续保留，便于版本对照。

## Quick start

```bash
cd backend && \
ENVOPS_SECURITY_TOKEN_SECRET=change-this-token-secret-2026-1234 \
ENVOPS_CREDENTIAL_PROTECTION_SECRET=change-this-credential-secret-2026 \
mvn -pl envops-boot spring-boot:run
```

```bash
cd frontend && pnpm install && pnpm dev
```

默认体验账号：`envops-admin / EnvOps@123`

补充说明：

- 后端默认地址：`http://127.0.0.1:8080`
- 前端默认地址：`http://localhost:9527`
- 两个后端安全相关环境变量都要求至少 32 个字符
- 后端启动时自动加载 `schema.sql` 与 `data.sql`

## Known limitations

- `home` 当前仍以平台概览页为主。
- `envops-exec` 当前仍是模块骨架。
- 流量 plugin 当前返回的是 skeleton 结果，适合联调和契约验证，不是完整生产流量切换能力。
- `supports*` 只能表示动作能力位，不能替代 `pluginStatus` 对实现完成度的表达。
- `SshRemoteExecutor` 当前 `detect` 仍是适配器级反馈，不是完整探测编排能力。
- `system/user` 当前已支持列表、新建、编辑，但仍不是完整身份治理后台。
- `frontend/.env.prod` 的 `VITE_SERVICE_BASE_URL` 仍为空，生产部署前需明确 API 接入方式。
