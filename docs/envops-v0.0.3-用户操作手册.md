# EnvOps / CMDB v0.0.3 用户操作手册

## 1. 手册说明

本文档面向平台使用者、联调人员和演示环境体验者，重点说明当前 `v0.0.3` 版本下已经可见、可操作的页面、菜单和典型流程。

适用对象主要包括：

- 超级管理员
- 运维管理员
- 发布工程师
- 测试 / 联调人员
- 演示环境体验者

> 说明：本文手册以当前 `frontend/`、`backend/` 的真实代码行为为准。对于尚未接入真实外部系统的能力，文中会明确标注边界。

---

## 2. 使用前准备

## 2.1 环境准备

如果你在本地体验当前版本，请先确认：

1. 后端已经启动，默认地址为 `http://127.0.0.1:8080`
2. 前端已经启动，默认地址为 `http://localhost:9527`
3. 浏览器可以访问前端页面
4. 当前环境已加载种子数据

## 2.2 默认体验账号

当前版本默认提供本地体验账号：

- 用户名：`envops-admin`
- 密码：`EnvOps@123`

该账号默认具备管理员权限，适合本地演示、联调和功能浏览。

## 2.3 使用前注意事项

1. 当前版本中，资产、检测、应用、任务、系统管理和流量控制已经接入真实接口。
2. 流量控制页面虽然可以操作真实接口，但 plugin 结果当前仍以 skeleton 反馈为主，适合演示和联调。
3. 发布任务页面当前已支持创建、审批、查询和处置。
4. 首页仍以平台概览为主。

---

## 3. 登录与进入系统

## 3.1 登录入口

打开浏览器后访问：

- `http://localhost:9527`

系统将进入登录页。

## 3.2 登录步骤

1. 输入用户名 `envops-admin`
2. 输入密码 `EnvOps@123`
3. 点击登录
4. 登录成功后，系统会自动加载用户信息和动态菜单
5. 跳转进入首页

## 3.3 登录后的系统行为

登录成功后，系统会自动：

- 保存登录 token
- 获取当前用户信息
- 加载常量路由与用户路由
- 根据权限显示菜单

如果后端未启动，或地址配置错误，登录后页面会提示请求失败。

当前版本常见一级菜单包括：

- 首页
- 资产中心
- 检测中心
- 应用中心

## 5. 首页

首页主要用于展示平台摘要信息和常见关注点，包括：

- 应用数量、主机数量、运行任务数量等摘要卡片
- 发布焦点和检测焦点
- 平台准备度说明

当前首页以平台概览展示为主，不直接提供复杂业务操作入口，也不作为真实业务数据看板使用。

## 6. 资产中心

资产中心是当前版本中已经接入真实接口的模块之一。

## 6.1 主机管理

路径：`资产中心 -> 主机管理`

当前可用能力：

- 查看主机列表
- 查看主机 IP、环境、集群、负责人、状态、最后心跳时间
- 新增纳管主机
- 手动刷新列表
- 对已有监控快照的主机进入指标详情

### 主机纳管操作步骤

1. 打开主机管理页面
2. 在左侧表单中填写：
   - 主机名
   - IP 地址
   - 环境
   - 集群名称
   - 归属团队
   - 状态
   - 最近心跳（可选）
3. 点击“纳管主机”
4. 创建成功后，页面会提示成功消息并刷新列表
5. 在右侧列表确认新主机已经出现

### 查看主机指标详情

1. 在主机列表中找到已有监控快照的主机
2. 点击“查看指标”
3. 跳转到“检测中心 -> 指标快照”页面查看该主机最新事实信息

## 6.2 凭据管理

路径：`资产中心 -> 凭据管理`

当前可用能力：

- 查看已有凭据列表
- 新增凭据
- 支持凭据类型：
  - SSH 密码
  - SSH 密钥
  - API Token

新增凭据操作步骤：

1. 打开凭据管理页面
2. 在表单中填写名称
3. 选择凭据类型
4. 根据需要填写用户名、密文内容、描述
5. 点击创建
6. 创建成功后，列表会自动刷新

使用注意事项：

- 名称和凭据类型是最基础的必填信息
- 凭据内容会进入后端保护逻辑，不会以明文方式直接展示为业务字段

## 6.3 分组管理

路径：`资产中心 -> 分组管理`

当前可用能力：

- 查看分组列表
- 查看分组描述与主机数量

当前页面主要用于浏览现有种子数据，不提供新增和编辑入口。

## 6.4 标签管理

路径：`资产中心 -> 标签管理`

当前可用能力：

- 查看标签列表
- 查看标签颜色和描述

当前页面主要用于浏览现有种子数据，不提供新增和编辑入口。

## 7. 检测中心

检测中心当前已接入真实接口，主要用于查看检测任务和主机快照。

## 7.1 即时检测

路径：`检测中心 -> 即时检测`

当前可用能力：

- 查看检测任务列表
- 查看检测目标、调度方式、最近执行时间、最近结果
- 新增检测任务
- 手动执行检测任务
- 刷新任务列表

### 新增检测任务步骤

1. 打开即时检测页面
2. 在左侧表单输入任务名称
3. 选择目标主机
4. 选择调度方式
5. 点击“创建任务”
6. 创建成功后，任务列表会刷新

### 执行检测任务步骤

1. 在任务列表中找到目标任务
2. 点击“执行”
3. 等待页面提示执行成功
4. 刷新后的列表会展示新的状态或最新执行结果

## 7.2 指标快照

路径：`检测中心 -> 指标快照`

当前可用能力：

- 选择主机
- 查看主机最新事实快照

适合在联调或演示时快速确认主机基础信息是否已经被采集。

## 9. 安装发布

安装发布当前已经具备任务创建、审批、查询、详情和任务处置能力。

## 9.1 发布任务列表

路径：`安装发布 -> 发布任务`

当前可用能力：

- 查看发布任务列表
- 按关键字、状态、任务类型、应用、环境、时间范围筛选
- 查看统计摘要
- 打开任务详情抽屉
- 打开创建任务抽屉

## 9.2 创建发布任务

1. 进入“安装发布 -> 发布任务”
2. 点击“创建”
3. 填写：
   - 任务名称
   - 任务类型
   - 环境
   - 应用
   - 版本
   - 批次策略与批次大小
   - 部署目录
   - SSH 用户
   - SSH 端口
   - 私钥路径
   - 远端基础目录
   - 回滚命令（可选）
   - 目标主机
4. 点击“创建”提交
5. 成功后系统会刷新列表，并自动打开该任务详情

## 9.3 发布任务审批

当前前端支持：

- 审批通过
- 审批驳回

操作步骤：

1. 在任务列表中找到待审批任务
2. 点击“批准”或“驳回”
3. 在抽屉中填写审批备注（可选）
4. 提交后刷新任务状态

## 9.4 发布任务详情

在任务列表中打开任意任务后，可查看：

- 总览信息
- 主机明细
- 执行日志

系统会根据任务状态自动刷新详情信息，便于观察任务变化。

## 9.5 当前支持的任务动作

对符合条件的任务，当前前端支持以下动作：

- 执行 `execute`
- 重试 `retry`
- 回滚 `rollback`
- 取消 `cancel`
- 批准 `approve`
- 驳回 `reject`

这些动作依赖后端当前任务状态和校验逻辑，不是所有任务都能看到所有按钮。

路径：`任务中心 -> 全部任务`

任务中心当前用于统一查看任务列表和风险情况。

## 10.1 当前可用能力

- 按关键字、状态、来源类型、任务类型、优先级筛选

## 10.2 典型使用方式

## 11. 流量控制

路径：`流量控制 -> 流量规则`

当前页面主要用于查看流量策略并执行预览、应用、回滚动作。

## 11.1 当前可用能力

- 查看流量策略列表
- 查看策略范围、比例、责任方、状态
- 执行 `preview`
- 执行 `apply`
- 在存在回滚令牌时执行 `rollback`
- 查看最近一次动作反馈

## 11.2 流量动作操作步骤

1. 打开“流量控制 -> 流量规则”
2. 在列表中找到目标策略
3. 根据按钮可用状态选择 `preview`、`apply` 或 `rollback`
4. 操作成功后，页面会显示最近一次动作结果反馈
5. 点击刷新，确认策略状态是否变化

## 11.3 使用注意事项

- 不是所有策略都能执行全部动作，按钮是否可点取决于 plugin 能力位和回滚令牌状态
- `supportsPreview`、`supportsApply`、`supportsRollback` 表示动作是否暴露，不等于真实网关已经打通
- 当前 `NGINX` / `REST` plugin 仍以 skeleton 消息为主，更适合作为联调、演示和契约验证能力

当前页面已支持真实用户管理基础操作。

## 12.1 当前可用能力

- 查看系统用户列表
- 查看用户摘要卡片
- 新增用户
- 编辑用户
- 维护团队、登录方式、状态、角色

## 12.2 新增用户步骤

1. 打开“系统管理 -> 用户管理”
2. 点击“新增用户”
3. 填写：
   - 用户名
   - 密码
   - 手机号
   - 团队
   - 登录方式
   - 状态
   - 角色
4. 点击“保存”
5. 成功后系统会提示并刷新列表

## 12.3 编辑用户步骤

1. 在用户列表中找到目标用户
2. 点击“编辑”
3. 修改手机号、团队、登录方式、状态、角色等信息

## 12.4 使用注意事项

## 13. 典型操作流程

## 13.1 本地体验完整路径

推荐按以下顺序体验当前版本：

1. 启动后端
2. 启动前端
3. 使用 `envops-admin / EnvOps@123` 登录
4. 在资产中心纳管一台主机
5. 在检测中心为该主机新增一个检测任务并执行
6. 查看主机指标快照
7. 在应用中心查看应用、版本、安装包和模板
8. 在安装发布创建一个任务，并根据状态执行审批
9. 查看任务详情、主机明细和日志
10. 在系统管理页面新增或编辑用户
11. 在流量控制页面执行一次 `preview` 或 `apply`

## 13.2 主机纳管流程

1. 打开“资产中心 -> 主机管理”
2. 填写主机名、IP、环境、集群、归属团队和状态
3. 按需填写最近心跳
4. 点击“纳管主机”
5. 确认新主机已经出现在列表中

## 13.3 检测任务创建与执行流程

1. 打开“检测中心 -> 即时检测”
2. 输入任务名称
3. 选择目标主机
4. 选择调度方式
5. 点击“创建任务”
6. 在列表中找到任务并点击“执行”
7. 观察状态和最近结果变化

## 13.4 发布任务流程

1. 打开“安装发布 -> 发布任务”
2. 点击“创建”并填写任务表单
3. 提交后查看新任务
4. 对待审批任务执行批准或驳回
5. 查看详情、主机明细和日志
6. 根据状态执行 `execute`、`retry`、`rollback` 或 `cancel`

## 13.5 系统用户管理流程

1. 打开“系统管理 -> 用户管理”
2. 点击“新增用户”创建新用户
3. 回到列表确认用户已出现
4. 点击“编辑”更新角色、状态或登录方式

## 13.6 流量动作流程

1. 打开“流量控制 -> 流量规则”
2. 选择一条策略
3. 先执行 `preview`
4. 再根据需要执行 `apply`
5. 若策略已有回滚令牌，可执行 `rollback`
6. 根据页面提示查看最近一次动作结果

## 14. 风险与限制说明

1. 首页当前多为概览展示页。
2. 生产环境 API 地址尚未在前端 `.env.prod` 中填写完成。
3. 当前默认账号和种子数据仅适用于本地体验，不应用于正式环境。
4. 流量控制页面虽然可操作，但 plugin 结果仍不是完整生产执行结果。
5. `RemoteExecutor` 属于发布执行侧适配器，不属于流量 plugin 范畴。
6. 系统用户页当前已可操作，但还不是完整账号治理后台。
