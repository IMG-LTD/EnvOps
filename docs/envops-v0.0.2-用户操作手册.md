# EnvOps / CMDB v0.0.2 用户操作手册

## 1. 手册说明

本文档面向平台使用者、联调人员和演示环境体验者，重点说明当前 `v0.0.2` 版本下已经可见、可操作的页面、菜单和典型流程。

适用对象主要包括：

- 超级管理员
- 运维管理员
- 发布工程师
- 测试 / 联调人员
- 演示环境体验者

> 说明：本文手册以当前 `frontend/`、`backend/` 的真实代码行为为准。对于尚未接入真实接口的页面，文中会明确标注为“展示页”或“预留页”。

---

## 2. 使用前准备

## 2.1 环境准备

如果你在本地体验当前版本，请先确认：

1. 后端已经启动，默认地址为 `http://127.0.0.1:8080`
2. 前端已经启动，默认地址为 `http://localhost:9527`
3. 浏览器可以访问前端页面
4. 当前环境已加载种子数据

## 2.2 默认体验账号

当前版本默认提供本地体验账号：

- 用户名：`envops-admin`
- 密码：`EnvOps@123`

该账号默认具备管理员权限，适合本地演示、联调和功能浏览。

## 2.3 使用前注意事项

1. 当前版本中，部分页面已经接入真实业务接口，部分页面仍是展示页。
2. 流量控制、系统管理等菜单当前更适合作为功能理解入口，不建议按“完整生产能力”理解。
3. 发布任务页面当前支持查询和处置，不提供任务新建页面。

---

## 3. 登录与进入系统

## 3.1 登录入口

打开浏览器后访问：

- `http://localhost:9527`

系统将进入登录页。

## 3.2 登录步骤

1. 输入用户名 `envops-admin`
2. 输入密码 `EnvOps@123`
3. 点击登录
4. 登录成功后，系统会自动加载用户信息和动态菜单
5. 跳转进入首页

## 3.3 登录后的系统行为

登录成功后，系统会自动：

- 保存登录 token
- 获取当前用户信息
- 加载常量路由与用户路由
- 根据权限显示菜单

如果后端未启动，或地址配置错误，登录后页面会提示请求失败。

## 4. 菜单概览

当前版本常见一级菜单包括：

- 首页
- 资产中心
- 检测中心
- 应用中心
- 安装发布
- 任务中心
- 流量控制
- 系统管理

从当前代码实际落地情况看：

- **真实业务菜单**：资产中心、检测中心、应用中心、安装发布、任务中心
- **展示/预留菜单**：首页、流量控制、系统管理

---

## 5. 首页

首页主要用于展示平台摘要信息和常见关注点，包括：

- 应用数量、主机数量、运行任务数量等摘要卡片
- 发布焦点和检测焦点
- 平台准备度说明

当前首页以平台概览展示为主，不直接提供复杂业务操作入口，也不作为真实业务数据看板使用。

---

## 6. 资产中心

资产中心是当前版本中已经接入真实接口的模块之一。

## 6.1 主机管理

路径：`资产中心 -> 主机管理`

当前可用能力：

- 查看主机列表
- 查看主机 IP、环境、集群、负责人、状态、最后心跳时间
- 手动刷新列表

典型使用方式：

1. 打开主机管理页面
2. 查看主机总数、在线数、待维护数摘要
3. 在列表中核对主机环境和状态
4. 点击刷新，重新获取后端数据

## 6.2 凭据管理

路径：`资产中心 -> 凭据管理`

当前可用能力：

- 查看已有凭据列表
- 新增凭据
- 支持凭据类型：
  - SSH 密码
  - SSH 密钥
  - API Token

新增凭据操作步骤：

1. 打开凭据管理页面
2. 在表单中填写名称
3. 选择凭据类型
4. 根据需要填写用户名、密文内容、描述
5. 点击创建
6. 创建成功后，列表会自动刷新

使用注意事项：

- 名称和凭据类型是最基础的必填信息
- 凭据内容会进入后端保护逻辑，不会以明文方式直接展示为业务字段

## 6.3 分组管理

路径：`资产中心 -> 分组管理`

当前可用能力：

- 查看分组列表
- 查看分组描述与主机数量

当前页面主要用于浏览现有种子数据，不提供新增和编辑入口。

## 6.4 标签管理

路径：`资产中心 -> 标签管理`

当前可用能力：

- 查看标签列表
- 查看标签颜色和描述

当前页面主要用于浏览现有种子数据，不提供新增和编辑入口。

---

## 7. 检测中心

检测中心当前已接入真实接口，主要用于查看检测任务和主机快照。

## 7.1 即时检测

路径：`检测中心 -> 即时检测`

当前可用能力：

- 查看检测任务列表
- 查看检测目标、调度方式、最近执行时间、最近结果
- 查看成功数与关注项统计
- 刷新任务列表

典型使用方式：

1. 打开即时检测页面
2. 查看检测任务总数和成功率
3. 关注 `warning`、`timeout`、`failed` 状态任务
4. 刷新列表，确认最新状态

## 7.2 指标快照

路径：`检测中心 -> 指标快照`

当前可用能力：

- 选择主机
- 查看主机最新事实快照

适合在联调或演示时快速确认主机基础信息是否已经被采集。

---

## 8. 应用中心

应用中心是当前版本中功能最完整的模块之一，已经具备增删改查和上传能力。

## 8.1 应用定义

路径：`应用中心 -> 应用定义`

当前可用能力：

- 查看应用列表
- 查看应用详情
- 新增应用
- 编辑应用
- 删除应用

典型操作步骤：

1. 进入应用定义页
2. 点击“新增”
3. 填写应用编码、名称、应用类型、运行时类型、部署方式、默认端口等信息
4. 提交后返回列表
5. 点击某条记录可查看详情

## 8.2 版本管理

路径：`应用中心 -> 版本管理`

当前可用能力：

- 按应用查看版本列表
- 新增版本
- 编辑版本
- 删除版本
- 关联安装包、配置模板、脚本模板

该页面是应用建模和后续发布衔接的重要入口。

## 8.3 安装包管理

路径：`应用中心 -> 安装包管理`

当前可用能力：

- 查看安装包列表
- 上传安装包
- 删除安装包

上传安装包步骤：

1. 填写安装包名称
2. 选择包类型
3. 选择存储类型
4. 选择文件
5. 点击上传
6. 上传成功后刷新列表

## 8.4 配置模板

路径：`应用中心 -> 配置模板`

当前可用能力：

- 查看模板列表
- 新增模板
- 编辑模板
- 删除模板

## 8.5 脚本模板

路径：`应用中心 -> 脚本模板`

当前可用能力：

- 查看模板列表
- 新增模板
- 编辑模板
- 删除模板

---

## 9. 安装发布

安装发布当前已经具备任务查询、详情和任务处置能力，但不提供任务新建入口。

## 9.1 发布任务列表

路径：`安装发布 -> 发布任务`

当前可用能力：

- 查看发布任务列表
- 按关键字、状态、任务类型、应用、环境、时间范围筛选
- 查看统计摘要
- 打开任务详情抽屉

## 9.2 发布任务详情

在任务列表中打开任意任务后，可查看：

- 总览信息
- 主机明细
- 执行日志

系统会根据任务状态自动刷新详情信息，便于观察任务变化。

## 9.3 当前支持的任务动作

对符合条件的任务，当前前端支持以下动作：

- 执行 `execute`
- 重试 `retry`
- 回滚 `rollback`
- 取消 `cancel`

这些动作依赖后端当前任务状态和校验逻辑，不是所有任务都能看到所有按钮。

## 9.4 当前不包含的功能

当前前端**不包含**：

- 新建发布任务页面
- 发布任务审批通过页面
- 发布任务驳回页面

如果需要演示完整审批链路，需要先通过后端接口或测试数据准备对应任务。

---

## 10. 任务中心

路径：`任务中心 -> 全部任务`

任务中心当前用于统一查看任务列表和风险情况。

## 10.1 当前可用能力

- 按关键字、状态、来源类型、任务类型、优先级筛选
- 分页查看任务列表
- 查看队列中任务、运行中任务、SLA 风险任务摘要
- 从任务中心跳转到发布任务详情

## 10.2 典型使用方式

1. 打开任务中心
2. 通过筛选条件缩小范围
3. 查看运行中或失败任务
4. 点击跳转到发布任务详情页进一步查看日志与主机明细

---

## 11. 流量控制

路径：`流量控制 -> 流量规则`

当前页面主要用于展示流量控制相关概念和示例数据，包括：

- 流量策略
- 范围
- 权重比例
- 责任方
- 状态

当前版本中，这一页更适合作为功能预览和术语说明入口，不代表已经具备真实生产流量切换能力。

---

## 12. 系统管理

路径：`系统管理 -> 用户管理`

当前页面主要展示用户、角色、团队、登录方式、最后登录时间等静态示例信息。

它更适合作为系统管理模块的页面骨架，不应视为完整的用户管理后台。

---

## 13. 典型操作流程

## 13.1 本地体验完整路径

推荐按以下顺序体验当前版本：

1. 启动后端
2. 启动前端
3. 使用 `envops-admin / EnvOps@123` 登录
4. 查看首页与菜单是否正常加载
5. 进入资产中心查看主机并新增一条凭据
6. 进入检测中心查看检测任务与主机快照
7. 进入应用中心查看应用、版本、安装包和模板
8. 进入安装发布查看示例任务详情、主机明细和日志
9. 进入任务中心查看聚合任务列表

## 13.2 凭据新增流程

1. 打开“资产中心 -> 凭据管理”
2. 输入名称
3. 选择凭据类型
4. 填写用户名、密文和描述
5. 点击创建
6. 查看新增结果是否出现在列表中

## 13.3 应用建模流程

1. 打开“应用中心 -> 应用定义”新增应用
2. 打开“版本管理”给该应用新增版本
3. 打开“安装包管理”上传安装包
4. 打开“配置模板”和“脚本模板”维护模板内容
5. 完成一个可用于后续发布的应用版本准备

## 13.4 发布任务查看流程

1. 打开“安装发布 -> 发布任务”
2. 按条件筛选任务
3. 选择任务并打开详情
4. 查看总览、主机明细、执行日志
5. 对满足条件的任务进行执行、重试、回滚或取消

## 13.5 任务风险排查流程

1. 打开“任务中心 -> 全部任务”
2. 关注运行中、失败或高优先级任务
3. 根据需要跳转到发布任务详情
4. 查看日志和主机明细定位问题

---

## 14. 风险与限制说明

1. 首页、流量控制、系统管理当前多为展示页。
2. 发布任务页面没有新建和审批入口。
3. 生产环境 API 地址尚未在前端 `.env.prod` 中填写完成。
4. 当前默认账号和种子数据仅适用于本地体验，不应用于正式环境。

---

## 15. 结论

当前 `v0.0.2` 已经能够支持一条比较完整的本地体验链路：

- 登录
- 动态菜单加载
- 资产浏览与部分录入
- 检测任务与快照查看
- 应用建模
- 发布任务查询与处置
- 任务中心统一查看
